# 网站发布清单
**版本**：v1.0 | 2026-04-19  
**用途**：每次发布新页面或新文章前，逐项检查

---

## 一、页面通用检查项

### 基础 SEO

- [ ] **标题是否清楚**  
  页面 `<title>` 是否包含核心关键词 + 赞诺品牌词？  
  格式建议：`关键词描述 · Zeno`

- [ ] **关键词是否明确**  
  是否在写文章前确定了 2-3 个目标关键词？  
  关键词是否自然出现在标题、副标题、正文前两段？

- [ ] **是否有页面摘要（Meta Description）**  
  Meta description 是否在 120-160 字之间？  
  是否包含核心关键词？  
  是否像人说话，不像 SEO 填充句？

- [ ] **URL 是否干净**  
  URL 是否使用拼音 slug？  
  是否不含日期、特殊符号、多余参数？

### 内容质量

- [ ] **是否有清晰的立场或判断**  
  这篇文章/这个页面，有没有表达赞诺的真实观点？  
  读者读完能不能知道「赞诺是怎么想的」？

- [ ] **是否有内链**  
  文章/页面内是否有 2-3 个内链指向相关文章、专题、资料或服务？  
  内链文字是否自然，不是硬凑的「点这里」？

- [ ] **是否有服务入口**  
  文章结尾是否有「如果你需要更具体帮助」的服务引导？  
  服务入口是否与文章内容相关（不是每篇都接同一个服务）？

- [ ] **是否有资料入口**  
  文章结尾是否有关联资料的领取引导？  
  资料领取文案是否自然，不强推？

### GEO / AI 引用适配

- [ ] **是否有 FAQ 区块**  
  文章是否包含 3-5 个问答格式的 FAQ？  
  FAQ 问题是否覆盖读者可能在搜索引擎或 AI 里问的真实问题？

- [ ] **是否有适合 AI 引用的简明答案**  
  文章开头或 Hero 区块是否有一段 50-100 字的直接回答？  
  回答是否清晰、可独立引用，不依赖上下文理解？

- [ ] **是否适合 AI 引用**  
  整体内容信息密度是否够高？  
  是否避免了大量空话、情绪词、口水话？

### 人格与表达

- [ ] **是否保留赞诺的人格表达**  
  文章/页面是否有清晰的第一人称观点？  
  是否避免了「大家」「我们」等模糊主语？

- [ ] **是否避免营销腔**  
  是否没有「全网最全」「保姆级」「必看」「宝藏」等表达？  
  服务介绍是否诚实写了「不适合谁」？

- [ ] **是否避免空泛鸡汤**  
  文章是否有具体的信息、判断、方法，而不只是情绪性的鼓励？

- [ ] **是否没有编造真实案例**  
  不确定的数据和案例是否用【待补充】标注？  
  没有用「一个客户说...」却没有真实来源的故事？

## 二、文章专项检查项

| 检查项 | 说明 |
|---|---|
| 文章字数 | 建议 1500-3000 字（SEO 文章建议 2000+）|
| 标题层级 | 使用 H2/H3，不跳级 |
| 段落长度 | 每段不超过 6 行，便于移动端阅读 |
| 首段是否清晰 | 开头 2-3 句直接切入主题，不废话 |
| 文末是否收尾 | 结尾是否有清晰的落点，不是随机截止 |
| 图片 alt | 如有图片，是否有描述性 alt 文字？ |
| 结构化数据 | 是否添加了 JSON-LD（Article / FAQ） |

## 三、上线前最终检查

- [ ] 移动端是否显示正常（用手机打开预览）
- [ ] 标题和摘要是否在微信分享卡片里显示正常
- [ ] 所有内链是否能正常跳转（无 404）
- [ ] 资料领取流程是否通畅（公众号关键词回复是否设置）
- [ ] 服务咨询入口是否有效（微信号 / 邮箱是否可以联系）
