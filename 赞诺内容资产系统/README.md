## 一、这个系统是干什么的

这不是一个博客管理系统，也不是笔记本。

这是赞诺的**内容生产操作系统**。

每一篇文章、每一个资料、每一个观点，都是一个内容资产单元。  
一个资产单元，可以变成网站文章、公众号推文、小红书帖子、抖音视频、YouTube 视频。  
每个平台版本，都服务于不同目的：搜索、信任、分发、转化。

**核心逻辑：** 写一次，用多次。观点是我的，表达适应平台。

## 三、新手如何开始

### 第一步：读 CLAUDE.md（5分钟）
了解整个系统的规则和角色分工。

### 第二步：填写 00_定位系统/个人资料包.md（30分钟）
把你的真实经历、风格、价值观填进去。这是整个内容系统的根基。

### 第三步：看第一个内容资产单元（10分钟）
打开 `04_内容资产单元/2026-装修预算为什么总超/生产单.md`，了解一篇内容是如何从选题变成生产计划的。

### 第四步：补充真实案例（随时）
打开 `03_素材证据库/`，把你记得的工地案例、客户问题填进对应模板。这是整个系统最有价值的原材料。

### 第五步：开始写第一篇
用 `06_提示词库/母文写作提示词.md` 里的提示词，配合生产单，开始写母文。

## 四、一篇内容如何从灵感变成多平台资产

```
触发（客户问了一个问题，工地看到一个现象）
    ↓
记录到 01_灵感收集箱/
    ↓
在 02_选题地图/ 对应主轴下添加选题
    ↓
在 04_内容资产单元/ 新建文件夹，复制 _模板/ 里的文件
    ↓
填写 生产单.md（定义观点、受众、素材需求）
    ↓
补充 03_素材证据库/（填入真实案例和细节）
    ↓
写 母文.md（观点源代码）
    ↓
改写 网站版.md（SEO/GEO优化）
改写 公众号版.md（信任表达）
改写 小红书版.md（分发钩子）
改写 抖音口播版.md（短视频）
    ↓
发布各平台
    ↓
填写 07_发布复盘/发布记录.md
填写 复盘.md
```

## 五、各平台的角色

| 平台 | 角色 | 核心目的 |
|---|---|---|
| 网站（zenoaihome.com）| 长期母站 | SEO 排名、GEO 引用、服务转化 |
| 公众号（Zeno AI装修笔记）| 信任入口 | 建立读者信任、私域积累 |
| 小红书 | 分发入口 | 新用户发现、流量入口 |
| 抖音 | 分发入口 | 更大范围曝光 |
| YouTube | 海外/长内容 | 长期积累，视频资产 |

**核心原则：** 网站是永久资产，其他平台都是分发渠道。  
所有内容最终要有网站版本，因为只有网站是你完全控制的。

## 六、旧系统如何处理

旧系统文件已保留在 `09_网站母站/` 下（网站相关）和 `99_旧系统归档/`（其他旧文件）。

**规则：**
- 旧系统只做参考，不复制内容到新流程
- 新内容全部按新流程生产
- 旧系统文件不删除，只归档

## 七、系统核心规则（重申）

1. 一个选题 = 一个内容资产单元（一个文件夹）
2. AI 是执行层，不是人格层
3. 先定观点，再写文章
4. 母文是源代码，平台版本是不同表达
5. 网站是长期母站，公众号是信任表达，小红书/抖音是分发入口
6. 每篇内容兼顾：人格资产 + 搜索资产 + AI引用资产 + 转化资产
7. 系统追求低摩擦，不追求文件多
8. 旧系统只做归档参考
