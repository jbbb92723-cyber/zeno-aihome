> 版本：v1.0 · 2026-04-24  
> 作者：品牌架构 / 产品 / 信息架构 / 商业战略综合判断  
> 决策人：Zeno

## 一、为什么现阶段默认中文主站更合适

### 1.1 用户在哪，首页就朝向哪

Zeno 的核心用户画像：

- **直接客户**：南宁本地、广西及全国中文互联网上搜索装修避坑、报价审核、预算模板的业主。
- **内容读者**：公众号、小红书、抖音来源的中文用户。
- **潜在付费用户**：需要报价审核服务、资料包、社群的人。

这些人 100% 是中文用户。他们第一次访问 zenoaihome.com 时如果看到英文首页，会在 3 秒内离开——不是因为他们不懂英文，而是因为**英文首页传递的信号是"这不是给我的"**。

### 1.2 转化链路依赖中文生态

Zeno 现阶段最重要的转化路径：

```
内容（公众号/小红书/抖音） → 网站文章 → 资料领取 → 微信添加 → 服务咨询 → 付费
```

这条路径的每一步都在中文生态里。英文首页无法承接这条链路。

### 1.3 品牌信任尚未在英文世界建立

中文世界里，Zeno 已经有 50 篇原创文章、5 大专题、资料库、报价审核服务——这些构成了**可感知的内容资产**。

英文世界里，Zeno 还是一个陌生名字。如果直接用英文首页做默认入口，等于把已经建立的中文信任感弃用，换成一个零信任的英文空壳。

**结论：默认中文首页是保护已有资产、服务核心用户、维护转化能力的必选项。**

## 二、中文主站的任务是什么

中文主站是 Zeno 的**信任站 + 转化站 + 服务站**。

中文主站的 KPI：**资料领取数、微信添加数、报价审核付费数、文章阅读量**。

## 四、为什么不能直接做"全英文替换中文"

五个致命问题：

### 4.1 丢失已有中文 SEO 资产
50 篇中文文章已经有了 Google 和 Baidu 的索引。整站换英文 = 所有中文 URL 变 404 或重定向混乱 = SEO 灾难。

### 4.2 中断微信生态转化链
公众号文章里的网站链接、小红书引流、二维码背后的落地页——全部指向中文页面。换成英文后这些链路全部断裂。

### 4.3 英文内容量不足以支撑全站
目前英文只有几个页面的雏形。用它替代 50 篇中文全文 + 资料库 + 服务页 + 联系页 = 空壳网站。

### 4.4 核心用户不会看英文站
你的付费用户是中文装修业主。他们不需要英文。

### 4.5 品牌叙事需要分场景
中文语境下 Zeno 是"十六年装修师傅，南宁，现实派长期主义者"。  
英文语境下 Zeno 是"a renovation practitioner building long-term living philosophy with AI"。  
两种叙事的语感、信息密度、读者期待完全不同。机翻 = 两边都做不好。

## 五、为什么也不能只做中文、不做英文

五个战略损失：

### 5.1 Google 英文搜索是最大的免费流量池
Zeno 的选题（长期主义、装修哲学、AI + 传统行业升级）在英文世界的搜索量远大于中文。不做英文 = 放弃全球最大的有机流量源。

### 5.2 AI 搜索引擎偏好英文内容
Perplexity、ChatGPT Search、Gemini 的知识库以英文为主。如果 Zeno 的观点只存在于中文，AI 引擎几乎不会引用。

### 5.3 工具产品天然无国界
排版工具 (md2wechat) 虽然服务中文用户，但未来的 Prompt Playground、Checklist 工具完全可以国际化传播。工具的增长杠杆在英文世界更大。

### 5.4 品牌天花板受限
纯中文站的品牌天花板 = 中文互联网。加上英文入口后，品牌的可引用性、可传播性、可合作性都会数量级提升。

### 5.5 "从中国工地出发"本身就是英文世界的差异化
英文世界充满了硅谷式"AI 生产力"叙事。一个**真正从工地走出来的人**讲 AI + 长期主义，这个故事在英文世界非常稀缺——不做英文就浪费了这个天然差异化。

## 六、最优方案：中文默认 + /en 英文入口

```
zenoaihome.com         → 中文主站（默认）
zenoaihome.com/en      → 英文主站（独立入口）
```

### 为什么是这个方案

| 维度 | 中文默认 + /en | 全英文 | 双域名 | /zh + /en |
|------|---------------|--------|--------|-----------|
| 保护中文 SEO | ✅ 不动 | ❌ 全部丢失 | ⚠️ 分散 | ⚠️ 中文也要迁移 |
| 保护转化链路 | ✅ 不动 | ❌ 全断 | ⚠️ 双域名混乱 | ⚠️ 所有链接要加 /zh |
| 英文可独立扩展 | ✅ 独立路由 | ✅ | ✅ | ✅ |
| 开发复杂度 | 低 | 低 | 高（双部署） | 中（全部迁移） |
| Vercel 部署 | 1 个项目 | 1 个项目 | 2 个项目 | 1 个但需重构 |
| 域名成本 | 0 | 0 | +1 域名 | 0 |

**中文默认 + /en 是开发成本最低、风险最小、可扩展性最好的方案。**

### 为什么不需要 /zh

当前所有中文内容都在根路径下，所有平台链接、公众号链接、SEO 索引都指向根路径。迁移到 /zh 需要：
- 所有 URL 重写
- 所有外部链接失效
- 301 重定向维护
- 公众号历史文章链接修复

工作量巨大，收益为零。**根路径就是中文，/en 是英文。干净、清晰、无迁移成本。**

## 九、两者如何互不打架

### 9.1 URL 完全隔离

中文在根路径，英文在 /en。没有任何重叠。搜索引擎不会混淆。

### 9.2 导航独立

中文主站有中文导航栏（含"EN"切换入口）。  
英文主站有英文导航栏（含"中文"切换入口）。  
两套导航的栏目数量和结构可以不同。

### 9.3 内容不互译

英文站不是中文站的翻译。英文站是独立策划的内容。可以从中文选题里选材，但语言、结构、侧重点都要为英文读者重新写。

### 9.4 SEO 独立

- 中文页面的 `<html lang="zh-CN">`
- 英文页面的 `<html lang="en">`
- 后续可加 `hreflang` 标签指向对方版本（如果某篇文章中英都有）
- 但大部分页面不需要对应关系——因为两站内容不对等

### 9.5 CTA 不冲突

中文 CTA 导向微信生态。英文 CTA 导向国际化动作（工具试用、Newsletter、YouTube）。两者的转化终点不同，不会打架。

## 十、未来什么时候可以把英文站做重

### 做重的前提条件（满足 3 个即可启动）

1. **英文内容 ≥ 20 篇精选文章**——证明英文内容生产能力可持续
2. **Google 英文收录 ≥ 50 个页面**——证明搜索引擎已认可
3. **英文月独立访客 ≥ 1000**——证明有真实需求
4. **至少一个工具有 ≥ 500 月活跃用户**——证明工具传播有效
5. **YouTube/X 粉丝 ≥ 5000**——证明英文品牌有社交基础

满足上述任意 3 条时，可以考虑：
- 英文站增加更多内容
- 英文站增加完整的资料库
- 英文站做独立的服务体系（英文版报价审核、国际化咨询）
- 考虑是否需要独立域名（如 zeno.build 或 zenoai.home）

在此之前，英文站保持"轻量、高质量、可持续扩展"即可。

- "排版工具"保留在导航——它是差异化产品入口

```

- 比中文少了"资料库"和"服务"——英文版这两块还没内容支撑

- "Essays" 而非 "Blog"——更有品质感，匹配 Zeno 的定位
- "Services" 暂不放导航，在 About 页面底部做轻量入口即可

| 栏目 | 中文站 | 英文站 | 原因 |

| 文章(50篇) | ✅ 全量 | 精选 10-15 篇 | 英文内容宁少勿滥 |

## 第四部分：URL 策略

### 完整 URL 结构

**中文主站：**
```
/                          首页
/about                     关于我
/blog                      文章列表
/blog/[slug]               文章详情
/topics                    专题列表
/topics/[slug]             专题详情
/resources                 资料库
/services                  服务
/tools/md2wechat           排版工具
/contact                   联系
/login                     登录
/account                   账号
```

**英文主站：**
```
/en                        Home
/en/about                  About
/en/essays                 Essays
/en/essays/[slug]          Essay detail
/en/tools                  Tools index
/en/tools/[tool-slug]      Tool detail
/en/topics                 Topics
/en/topics/[slug]          Topic detail
/en/services               Services (lightweight)
/en/contact                Contact
```

### 关键决策说明

**1. 为什么根域名默认中文**

- 所有现有外部链接（公众号、小红书、Baidu 索引）都指向根路径
- 迁移成本为零
- 核心付费用户全是中文用户

**2. 为什么英文站放 /en**

- 子路径在同一个 Vercel 项目里，部署维护成本最低
- Next.js App Router 原生支持 `/en` 路由（已有 `app/en/` 目录）
- 不需要额外域名、DNS、证书
- 可以共享组件、样式、认证系统

**3. 是否需要 /zh**

**不需要。** 原因：
- 现有 50+ 篇文章全在根路径，迁移到 /zh 需要大量 301 重定向
- 所有外部链接（公众号/小红书/Baidu）都指向根路径
- /zh 对当前用户没有任何体验改善
- 如果未来某天需要做严格的 i18n 框架，可以用 Next.js 中间件做透明路由映射，不需要改 URL

**4. 中英文切换如何处理**

现有 Header 已实现 `LanguageSwitcher` 组件：
- 在中文页面显示 "EN"，点击跳转 `/en` + 当前路径
- 在英文页面显示 "中文"，点击跳转去掉 `/en` 前缀的路径
- 如果英文没有对应页面，跳转到 `/en` 首页即可

**5. 如何避免 SEO 混乱**

- 中文页面 `<html lang="zh-CN">`——已实现
- 英文页面需要在 layout 中设置 `<html lang="en">`——**需要改造英文 layout**
- 每个页面都有独立的 `title` 和 `description`——已实现
- 中英文页面的 `canonical` 各自独立，不互指

**6. hreflang 是否需要**

**目前不需要。**

hreflang 在以下场景才有意义：
- 同一篇内容有中英两个版本
- 你希望 Google 根据用户语言推荐不同版本

目前中英文站内容不对等（50 篇中文 vs 10-15 篇英文精选），大部分页面没有对应关系。等英文站内容量上来后，再对有对应关系的文章加 hreflang。

**优先级：工具试用 > 精选阅读 > Newsletter 订阅 > 社交关注**

### CTA 分工原则

1. **微信二维码不出现在英文站**——国际用户不用微信，二维码会降低专业感
2. **Newsletter 只在英文站推**——中文用户习惯公众号，不习惯邮件订阅
3. **报价审核服务只在中文站推**——服务面向中文业主
4. **工具 CTA 在英文站权重更高**——工具是国际传播的核心杠杆

## 第六部分：Grok 提议评估

### A. 资料库从 gated list 升级成 value experience layer

| 子项 | 结论 | 维度分析 |
|------|------|----------|
| 预算模板做在线交互版 | **可以做，但不急** | 商业回报中等（提升体验但不直接变现）、开发复杂度中等、当前 PDF/文档版已可用。→ 第二阶段。更适合**中文主站**。 |
| 验收清单做可勾选网页版/PWA | **可以做，但不急** | 商业回报中等、开发复杂度低-中、用户需求待验证。→ 第二阶段。更适合**中文主站**，也可做英文版。 |
| AI 提示词包做 Prompt Playground | **值得做，优先级中等** | 商业回报高（工具传播+品牌放大）、开发复杂度中等、适合**英文主站**做国际传播。→ 第二阶段优先。 |

**风险点**：交互版工具开发后需要持续维护，不要一口气做三个——先做一个验证。

### B. UI 现代化升级

| 子项 | 结论 | 维度分析 |
|------|------|----------|
| Shadcn UI | **暂时不做** | 现有 Tailwind 组件足够用，引入 Shadcn 需要全站组件迁移，收益不明显。 |
| Tailwind 4 | **暂时不做** | Tailwind 3 → 4 有 breaking changes，现阶段不值得为了版本号升级。 |
| Lucide Icons | **可以做，低优先** | 成本低、收益小。→ 有空顺手做。 |
| 暗黑模式 | **第二阶段** | 体验提升但不影响转化。→ 第二阶段。 |
| 现代产品感品牌站 | **⚠️ 谨慎** | 你明确要求"不要改首页到 SaaS 官网"——现有设计风格克制、有辨识度。不要追 SaaS 风。 |

**风险点**：UI "现代化"容易变成"SaaS化"——Zeno 的品牌是"工地里出来的真实人"，不是硅谷创业公司。视觉风格要保持克制和人文感。

### C. Hero + CTA 强化

| 子项 | 结论 | 维度分析 |
|------|------|----------|
| 增加强信任信号（数字、案例） | **可以做，但要真实** | 已有的数字（16 年经验、50 篇文章）可以用。但不要编造"服务 XXXX 客户"这种无法核实的数字。→ 第一阶段，仅限已有真实数据。 |
| 醒目二维码 | **中文站可以加，英文站不要** | 中文站首页或文章页可加浮动/底部二维码。英文站不放。 |
| 关注按钮 | **中文站做公众号关注按钮，英文站做 Newsletter** | 分场景执行。 |

**风险点**：信任信号一旦虚假或夸大，反噬极强。Zeno 的品牌建立在"真实"上，一个假数字会毁掉所有信任。

### D. 技术栈升级

| 子项 | 结论 | 维度分析 |
|------|------|----------|
| Next.js 15 | **暂时不做** | Next 14 完全够用，15 有 breaking changes（React 19 依赖等），升级成本高、收益低。 |
| React 19 | **暂时不做** | 同上。 |
| App Router | **已在用** ✅ | 无需改动。 |
| Server Actions | **可以做，按需引入** | 对表单场景有帮助。→ 第二阶段逐步引入。 |
| MDX | **第二阶段** | 对文章内嵌交互组件有价值。当前 Markdown 够用。 |
| Framer Motion | **暂时不做** | 动画不是当前优先级。 |
| ISR | **第二阶段** | 对性能和部署有帮助，但当前静态生成已满足需求。 |
| Web Vitals | **可以做** | 接 Vercel Analytics 即可，成本几乎为零。→ 第一阶段。 |
| Fuse.js / Orama | **第二阶段** | 搜索功能有价值，但当前 50 篇文章用导航即可。→ 英文站 20+ 篇后再做。 |

**风险点**：大版本升级是最常见的"技术债搬家"——花两周升级，换来的只是版本号变化，用户完全无感。不要做。

### E. 工具专区

| 子项 | 结论 | 维度分析 |
|------|------|----------|
| 预算计算器 | **第二阶段** | 有价值但开发量大、需要真实数据支撑。更适合**中文主站**。 |
| 实住派需求自测 | **第三阶段** | 概念有趣但用户需求未验证。 |
| AI 家装风格匹配器 | **第三阶段** | 开发复杂、需要大量训练数据/图片。现阶段不适合。 |

**风险点**：工具做出来容易，做"有人用"的工具很难。每个工具都要先验证需求再开发。

### F. 长期战略

| 子项 | 结论 | 维度分析 |
|------|------|----------|
| 站内 AI 助手 | **第三阶段** | 成本高（API 费用）、开发复杂、当前内容量不支撑有意义的 AI 对话。 |
| 社区化 | **第三阶段** | 需要持续运营投入，你一个人精力有限。 |
| 多渠道分发 | **已在做** | 公众号、小红书、抖音已有。YouTube/X 待启动。→ 第二阶段启动英文渠道。 |
| 数据飞轮 | **第二阶段启动** | 先接 Vercel Analytics，收集基础数据。→ 第二阶段基于数据优化内容策略。 |

**总结：Grok 的建议大约 70% 是好方向，但 80% 不适合现在做。**

## 第八部分：明确结论

### 1. 现阶段是不是应该默认中文首页？

**是。** 你的付费用户、内容读者、平台流量、微信生态全在中文。改成英文首页 = 主动放弃已有资产。

### 2. 英文站是不是应该作为 /en 入口？

**是。** /en 子路径在同一 Vercel 项目里，零额外成本，完美利用现有 Next.js App Router 架构。已有 `app/en/` 目录可直接扩展。

### 3. 中文主站和英文主站各自最重要的 KPI 是什么？

| 站点 | 核心 KPI |
|------|----------|
| 中文主站 | ① 资料领取数/月 ② 报价审核付费数/月 ③ 微信添加数/月 |
| 英文主站 | ① Google 英文收录页面数 ② 工具月活跃用户数 ③ Newsletter 订阅数 |

### 4. 第一阶段最值得优先做的 3 件事是什么？

1. **英文站导航 + layout 独立化**——让英文站有自己的 `<html lang="en">` 和独立导航，而不是复用中文 Header。这是所有英文站工作的基础。
2. **英文 Tools 页面搭建**——工具是英文站的核心差异化，也是最容易产生国际传播的内容。
3. **接入 Vercel Analytics**——零成本，但能让你第一次看到真实的流量数据，用数据指导后续决策。

### 5. 哪些 Grok 建议现在不能直接照搬？

| 建议 | 为什么不能照搬 |
|------|----------------|
| Next.js 15 + React 19 | 无用户感知收益，有 breaking changes 风险 |
| Shadcn UI | 全站组件迁移工作量大，现有方案足够 |
| Framer Motion | 动画不是当前优先级 |
| AI 助手 | 内容量不足、API 成本高、维护重 |
| 社区化 | 一个人精力有限，社区需要持续运营 |
| 预算计算器 / 风格匹配器 | 开发量大，用户需求未验证 |
| "现代产品感品牌站" | Zeno 的品牌是"真实人文"，不是"SaaS 酷感" |

### 6. 我的网站最终应该变成什么样的品牌模型？

```
Zeno 赞诺 品牌模型：

     中文主站（信任 + 转化 + 服务）
            zenoaihome.com
                 ↕
     英文主站（品牌 + 工具 + 放大）
            zenoaihome.com/en

     共同核心：
     "从工地走出来的现实派生活建造者"
     "从装修出发，聊居住、美学、人性、成长与 AI 时代的长期主义"

     中文 → 深度内容 + 服务变现 + 微信生态
     英文 → 品牌叙事 + 工具传播 + Google/AI/YouTube 增长

     一个双语的个人品牌 + 内容 + 工具平台

这不是一个"中文站 + 它的翻译版"。  
这是**两个有独立任务的站点，共享一个品牌和一套基础设施**。
