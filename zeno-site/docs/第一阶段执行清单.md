## 🔴 P0 — 必须做（本周内）

### 1. 英文站 layout 独立化

- **做什么**：修改 `app/en/layout.tsx`，让英文页面的 `<html>` 标签使用 `lang="en"`。目前英文站复用了中文的根 layout（`lang="zh-CN"`），这对 SEO 有负面影响。
- **为什么**：Google 根据 `lang` 属性判断页面语言。如果英文页面标记为 `zh-CN`，会导致英文内容无法正确进入 Google 英文索引。
- **难度**：低（改一个文件）
- **优先级**：🔴 P0
- **是否依赖开发**：是（需要改 Next.js layout）

---

### 2. 英文站独立导航组件

- **做什么**：创建 `components/HeaderEN.tsx`，导航项为：Home / About / Essays / Tools / Topics / Contact / 中文 / Log in。英文站 layout 使用 HeaderEN 替代中文 Header。
- **为什么**：中英文导航的栏目不同（英文没有"资料库""服务""排版工具"），复用中文导航会让英文用户看到无法使用的入口。
- **难度**：低-中（复制 Header.tsx 改导航数组）
- **优先级**：🔴 P0
- **是否依赖开发**：是

---

### 3. 英文 Tools 页面搭建

- **做什么**：创建 `app/en/tools/page.tsx`，做工具索引页。目前只有 md2wechat 排版工具，页面可以展示已有工具 + 预告"Prompt Playground coming soon"。
- **为什么**：Tools 是英文站的核心差异化和主要 CTA。没有 Tools 页面，英文站缺少可以国际传播的"产品"。
- **难度**：低
- **优先级**：🔴 P0
- **是否依赖开发**：是

---

### 4. 接入 Vercel Analytics

- **做什么**：在 Vercel 项目设置中开启 Analytics，或安装 `@vercel/analytics` 包并在根 layout 引入。
- **为什么**：你现在完全没有流量数据。没有数据 = 所有决策靠猜。接入 Analytics 成本几乎为零，但能立即开始积累数据。
- **难度**：极低（几行代码）
- **优先级**：🔴 P0
- **是否依赖开发**：是（极简）

---

### 5. 品牌命名确认 + Header 统一

- **做什么**：确认并统一以下命名规则：
  - 中文 Header logo 文字：`Zeno 赞诺`（已是现状 ✅）

  - `<title>` 中文默认：`Zeno 赞诺｜Zeno AI Home — ...`（已是现状 ✅）
  - `<title>` 英文默认：`Zeno — From renovation sites to long-term living`（已是现状 ✅）
- **为什么**：品牌命名统一是所有后续工作的基础。
- **难度**：极低（确认即可，可能不需要改代码）
