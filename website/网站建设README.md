# Zeno 个人网站信息架构（V1）
更新时间：2026-04-18

这不是传统装修公司官网，而是 Zeno 的个人信任中枢。  
核心表达不变：从装修出发，聊居住、美学、人性、成长与 AI 时代的长期主义。

## 1. 网站目标

1. 让第一次进入网站的人，在 10 秒内知道「你是谁、你在写什么、为什么值得继续看」。
2. 把分散在公众号、短视频、朋友圈里的内容沉淀成可检索、可回看、可持续积累的长期资产。
3. 建立稳定信任关系，而不是追求一次性转化。
4. 给咨询、合作、资料领取提供清晰入口，但不过度打扰阅读体验。

## 2. 目标用户

1. 正在装修或即将装修，但不想被“样板间逻辑”带偏的人。
2. 关心居住质量、审美与现实平衡的人。
3. 对判断力、成长、人性议题有长期兴趣的人。
4. 传统行业从业者里，正在学习 AI、想升级工作方式的人。

## 3. 一句话定位

从装修出发，聊居住、美学、人性、成长与 AI 时代的长期主义。

## 4. 首页结构

首页目标只有一个：让人愿意继续认识你。

1. 首屏（Hero）
定位句 + 简短自我说明 + 两个按钮：`看文章`、`了解我`。
2. 我主要写什么
用 5 个内容方向卡片说明写作边界，不做口号堆砌。
3. 最新文章
展示最近 3-6 篇，优先“代表你判断力”的内容，不按流量排序。
4. 重点专题
展示 3-4 个专题入口，强调“连续阅读价值”。
5. 资料库入口
放 1 个主资料和 2-3 个次资料，提供轻量领取方式。
6. 关于我（信任补全）
一段 80-120 字简介 + 一张真实照片 + `查看完整关于页`。
7. 联系入口
只放必要渠道：公众号、微信/邮箱、咨询入口。

## 5. 关于页结构

关于页不是履历墙，是信任页。

1. 我是谁
一句话身份定义 + 当前主要工作。
2. 我从哪里来
从装修一线到内容与系统化思考的路径。
3. 我为什么写这些
为什么从“装修问题”延伸到“居住、人性、成长、AI”。
4. 我的价值观
我相信什么、反对什么、坚持什么。
5. 我现在在做什么
当前内容方向、实践项目、可合作范围。
6. 如何联系我
明确可联系场景与不适合联系的事项，减少错配。

## 6. 文章分类

V1 固定 5 类，不追求越多越好。

1. 居住与装修：真实案例、流程、决策、避坑。
2. 美学与生活：空间审美、日常秩序、生活感。
3. 人性与判断：工地与生活里的决策机制、认知偏差。
4. 成长与长期主义：个人升级、节奏管理、长期投入。
5. AI 与新生产力：工具实践、工作流升级、行业迁移经验。

分类规则：
1. 每篇文章只归 1 个主分类。
2. 分类服务读者导航，不为栏目数量服务。

## 7. 专题规划

专题是“连续问题”的解法，不是简单标签聚合。

1. 实住派装修
核心问题：家是拿来住的，不是拿来拍样板照的。
2. 从工地看世界
核心问题：工程现场如何映射人的判断、协作与责任。
3. 长期主义生活
核心问题：如何在高噪音环境里保持长期选择。
4. 传统行业人如何用 AI 升级自己
核心问题：不被替代的关键能力如何重建。

## 8. 资料库规划

资料库定位：提供“拿来就能用”的工具，不做信息堆场。

V1 资料类型：
1. 装修预算模板
2. 验收清单
3. 报价审核清单
4. 实住派自检表
5. AI 内容/工作流提示词包

每份资料统一结构：
1. 解决什么问题
2. 适合谁
3. 怎么使用
4. 领取方式
5. 相关延伸阅读

## 9. 联系与转化路径

转化目标：先建立信任，再建立连接。

1. 内容入口：公众号/短视频/搜索进入文章页。
2. 信任建立：文章页内引导到专题与关于页。
3. 轻转化：通过资料领取留下联系方式。
4. 深转化：进入咨询或合作沟通。
5. 基础设施承接：通过 IDC Flare 提供域名解析、HTTPS 与基础加速/安全，保证访问稳定。

联系页仅保留三类入口：
1. 内容交流
2. 装修相关咨询
3. AI 工作流/内容系统交流

## 10. 第一版上线最小功能（MVP）

1. 基础页面可访问：首页、关于、文章、专题、资料库、联系。
2. 至少 5 篇种子文章上线（对应当前既定选题）。
3. 每个专题至少聚合 2 篇文章。
4. 资料库至少 3 份可领取资料。
5. 联系入口与资料领取流程可正常使用。
6. 全站支持基础 SEO 信息（标题、描述、Open Graph）。
7. 明确不接入登录系统，不配置第三方 OAuth 参数（应用名、应用主页、回调地址先留空）。
8. 接入 IDC Flare 基础能力：DNS、HTTPS、`www` 到主域名 301、基础缓存与安全规则。

## 11. 第二阶段再做的功能（登录、会员、资料权限、OAuth）

前提：持续更新稳定、已有明确复访人群后再启动。

1. 登录系统
支持邮箱登录或第三方登录，目标是降低重复填写成本。
2. 会员分层
区分公开内容、留资可见内容、会员内容。
3. 资料权限
按权限控制下载与查看范围，支持资料更新通知。
4. OAuth
接入微信/Google 等第三方身份体系，减少注册摩擦。
5. 个人中心
收藏、阅读记录、资料历史、订阅管理。
6. IDC Flare 深度接入
按业务情况细化缓存策略、边缘规则、性能监控与安全防护。

OAuth 启动条件（满足后再配置）：
1. 网站已有正式线上域名。
2. 登录系统已开发并可用。
3. 已有明确回调地址（如 `/api/auth/callback/...`）。
4. 再去第三方平台填写 `应用名`、`应用主页`、`回调地址` 等信息。

IDC Flare 接入原则（IF = IDC Flare）：
1. 先完成基础四件事：`zenoaihome.com` 解析、HTTPS 生效、主域名统一、可用性检查。
2. V1 以稳定优先：不启用激进缓存，确保内容更新后可及时看到。
3. 所有加速/安全规则可回滚，避免上线初期因配置过重影响访问与发布。

## 12. 不做什么

1. 不做传统装修公司官网，不走“案例堆砌 + 促销报价”路线。
2. 不做营销感很重的落地页，不用倒计时、夸张承诺、强迫转化。
3. 不急着做复杂登录与会员系统，先验证内容与信任是否成立。
4. 不为了功能而功能，任何新增都要服务长期内容资产与关系质量。
## 13. 网站内容风格规则

### 写作基本原则

1. **真实优先**：不写没有经历过的场景，不夸大、不美化，用具体细节代替抽象概念。
2. **克制表达**：不用感叹号堆砌情绪，不用"震惊/颠覆/逆袭"等词，情绪留在事实里。
3. **有立场**：不做"两方都对"式和稀泥表达，要有清晰判断，并给出判断依据。
4. **经历感**：装修、工地、客户、材料、审美、AI 是可以调用的真实经历，要让它们出现在文章里。
5. **读者视角**：每篇文章动笔前先回答"读这篇文章的人，在什么处境下、有什么困扰"，写作出发点始终是帮助而非表演。

### 禁忌词与禁忌句式

| 禁止使用 | 原因 |
|---|---|
| "干货满满" | 信息量靠内容本身体现，不靠自我标榜 |
| "建议收藏" "必看" "重磅" | 营销转发语气，破坏可信度 |
| "帮你少走弯路" | 过于居高临下，除非有具体场景支撑 |
| "逆袭" "躺赢" "财富自由" | 与长期主义气质不符 |
| "如果你也有这样的困惑" | 鸡汤起手式 |
| "关注我，下期继续分享" | 内容渠道语言，网站文章结尾不用 |
| 连续多个感叹号 | 降低阅读信任感 |
| 堆砌转折词："但是！然而！" | 人为制造阅读高潮，显得刻意 |

### 文章结构规范

每篇网站文章建议包含以下 frontmatter：
```yaml
---
title: 文章标题
date: YYYY-MM-DD
category: 居住与装修 | 美学与生活 | 人性与判断 | 成长与长期主义 | AI 与新生产力
tags: [标签1, 标签2]
excerpt: 100字以内摘要，不剧透结论，让人想读完
---
```

正文结构建议：
- **开头**：不用"大家好，我是 Zeno"，直接进入场景或问题。
- **主体**：用经历或案例带出判断，不用纯观点堆叠。
- **结尾**：以一句真实的总结或反问收尾，可自然引导领取资料或进入专题，不强迫留言转发。
- **长度**：1200-2000 字为主，不为凑字数而扩展，不为看起来精炼而过度删减。

### 页面文案规范

- 首页 Hero 不超过 30 字主标题 + 80 字副标题。
- 模块标题使用名词短语，避免动词命令式（"看文章" 改为 "文章"）。
- 按钮文案表达意图，不用废话："了解更多" → "了解我是谁"；"立即领取" → "领取资料"。
- 全站不出现价格、优惠、限时等词汇（v1 无商业化模块）。

## 14. 网站长期发展路线

### V1：个人信任中枢（当前目标，2026 Q2）

**核心任务**：把"我是谁、我在写什么、为什么值得信任"讲清楚。

- 上线基础6个页面：首页、关于我、文章、专题、资料库、联系
- 上线 5 篇种子文章（覆盖5个分类）
- 每个专题聚合 2+ 篇文章
- 资料库 3 份可领取资料
- 全站静态生成，部署在 Vercel
- 接入 IDC Flare 基础能力（DNS + HTTPS + 缓存）

**成功标准**：来访者能在 10 秒内判断是否继续，有 20% 的人会进入文章或资料库。

---

### V2：内容积累 + 轻量订阅（2026 Q3-Q4）

**触发条件**：文章积累到 20+ 篇，有稳定的复访流量。

- 接入邮件订阅（Resend 或 Buttondown）
- 接入全文搜索（Algolia 或本地 Fuse.js）
- 文章页新增阅读进度条、估计阅读时间
- 资料库扩充到 8+ 份，区分免费与留资可见
- 专题页展示文章数量与最近更新时间
- 新增 RSS Feed 支持
- 内容接入 Supabase 数据库（文章仍静态，但资料领取记录入库）

**成功标准**：有邮件订阅用户，每月新增 2+ 篇原创文章，资料库有持续领取记录。

---

### V3：会员权限 + 深度内容（2027+）

**触发条件**：有明确复访人群，有人主动问"在哪里看更多内容"。

- 登录系统（邮箱注册 + 第三方 OAuth）
- 内容权限分层：公开内容 / 留资可见 / 登录可见 / 付费内容
- 个人中心：收藏、阅读历史、资料下载记录
- 评论或留言系统（仅限登录用户）
- 可能的付费内容：深度专题、咨询产品、工作流模板
- 精细化 Cloudflare 边缘规则 + 性能监控

**成功标准**：有回头客，有口碑传播，内容资产本身能产生稳定价值。

---
